## 📋 Введение

**FAB Shield** можно начать использовать всего за несколько минут.

В этом руководстве показано, как быстро подключить базовую защиту к Node.js-приложению на Express.

Этот документ — минимальный путь к старту. Для более детальной настройки используйте полную документацию, примеры, разделы про плагины, метрики и расширенную конфигурацию.

## ⚡ Минимальная установка

### Шаг 1. Установка

Используя npm:

```bash
npm install @fab-orbita/shield
```

Используя yarn:

```bash
yarn add @fab-orbita/shield
```

Используя pnpm:

```bash
pnpm add @fab-orbita/shield
```

---

### Шаг 2. Подключение

```typescript
// app.ts или app.js

import express from 'express'
import { FABShield } from '@fab-orbita/shield'

const app = express()
const shield = new FABShield()

// Подключаем защиту
app.use(shield.middleware())

// Ваши маршруты
app.get('/', (req, res) => {
  res.json({
    message: 'Hello World!'
  })
})

app.listen(3000, () => {
  console.log('🚀 Сервер запущен на порту 3000')
})
```

---

### Шаг 3. Запуск

Для JavaScript:

```bash
node app.js
```

Для TypeScript:

```bash
npx tsx app.ts
```

Для разработки через nodemon:

```bash
nodemon app.js
```

## 📊 Проверка

### Быстрая проверка через curl

```bash
curl -I http://localhost:3000
```

В ответе должны появиться security-заголовки, например:

```text
Content-Security-Policy: ...
Strict-Transport-Security: ...
X-Frame-Options: SAMEORIGIN
X-Content-Type-Options: nosniff
Referrer-Policy: ...
```

1. Откройте приложение в браузере.
2. Откройте DevTools: `F12`.
3. Перейдите на вкладку **Network**.
4. Обновите страницу.
5. Выберите запрос.
6. Посмотрите **Response Headers**.

## 📝 Полный пример

```typescript
import express from 'express'
import { FABShield } from '@fab-orbita/shield'

const app = express()

app.use(express.json())

// Создаем защиту с настройками
const shield = new FABShield({
  // AI-защита
  ai: {
    enabled: true,
    anomalyDetection: true,
    threatPrediction: true,
    xssProtection: true,
    sqlInjectionProtection: true
  },

  // Мониторинг
  monitoring: {
    enabled: true,
    export: ['prometheus', 'json']
  },

  // Кастомные правила
  rules: [
    {
      path: '/api/*',
      method: 'POST',
      action: 'block',
      condition: (req) => !req.headers['x-api-key']
    }
  ],

  // Доверенные CDN
  trustedCDNs: [
    'https://cdnjs.cloudflare.com',
    'https://cdn.jsdelivr.net'
  ]
})

// Подключаем защиту до маршрутов
app.use(shield.middleware())

// API endpoint для метрик
app.get('/metrics', (req, res) => {
  const metrics = shield.getMetrics()
  res.json(metrics)
})

// Основной маршрут
app.get('/', (req, res) => {
  res.json({
    message: 'Hello World!',
    protected: true
  })
})

// Запуск
app.listen(3000, () => {
  console.log('🚀 FAB Shield активен!')
  console.log('🔒 Защита включена на порту 3000')
})

// Graceful shutdown
process.on('SIGTERM', () => {
  console.log('👋 Завершение работы...')
  process.exit(0)
})
```

## ❗ Что делать, если что-то пошло не так

### Проблема: ошибка при установке

Очистите npm cache:

```bash
npm cache clean --force
```

Установите пакет заново:

```bash
npm install @fab-orbita/shield
```

---

### Проблема: security-заголовки не появляются

Проверьте, что middleware подключен **до маршрутов**:

```typescript
// Правильно
app.use(shield.middleware())

app.get('/', (req, res) => {
  res.json({ ok: true })
})
```

Неправильный порядок:

```typescript
// Неправильно
app.get('/', (req, res) => {
  res.json({ ok: true })
})

app.use(shield.middleware())
```

---

### Проблема: слишком много логов

Уменьшите уровень логирования:

```typescript
import { FABShield } from '@fab-orbita/shield'

const shield = new FABShield({
  logging: {
    level: 'error'
  }
})
```

Возможные уровни:

| Уровень | Описание |
|:---|:---|
| `error` | Только ошибки |
| `warn` | Предупреждения и ошибки |
| `info` | Основные события |
| `debug` | Подробная отладка |
