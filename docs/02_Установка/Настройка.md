# ⚙️ Настройка FAB Shield

> Подробное руководство по конфигурации **FAB Shield**  
> Современного security-фреймворка для Node.js-приложений.

**Автор:** Фабрициус Владимир Николаевич

## 📋 Введение

**FAB Shield** предоставляет множество опций для тонкой настройки безопасности Node.js-приложений.

Правильная конфигурация позволяет:

- 🎯 оптимизировать защиту под конкретный проект;
- ⚡ улучшить производительность;
- 🔧 интегрироваться с существующими системами;
- 🛡️ включать только нужные security-модули;
- 📊 собирать метрики и события безопасности;
- 🔌 расширять возможности через плагины.

---

## 🏗️ Структура конфигурации

```typescript
interface ShieldConfig {
  // Основные настройки
  enabled?: boolean
  env?: 'development' | 'production' | 'test'

  // Security-заголовки
  headers?: HeaderConfig

  // CSP
  csp?: CSPConfig

  // AI-защита
  ai?: AIConfig

  // Мониторинг
  monitoring?: MonitoringConfig

  // Плагины
  plugins?: Plugin[]

  // Кастомные правила
  rules?: CustomRule[]

  // Логирование
  logging?: LoggingConfig

  // Производительность
  performance?: PerformanceConfig
}
```

## 🔒 Настройка Security-заголовков

### Базовая конфигурация

```typescript
import { FABShield } from '@fab-orbita/shield'

const shield = new FABShield({
  headers: {
    // Включить security-заголовки
    enabled: true,

    // Отключить конкретные заголовки
    disabled: ['X-Powered-By', 'X-XSS-Protection'],

    // Кастомные заголовки
    custom: {
      'X-Custom-Header': 'custom-value',
      'X-Security-Level': 'high'
    }
  }
})
```

---

### Настройка HSTS

```typescript
import { FABShield } from '@fab-orbita/shield'

const shield = new FABShield({
  headers: {
    hsts: {
      enabled: true,
      maxAge: 31536000,
      includeSubDomains: true,
      preload: true
    }
  }
})
```

**Что делает HSTS:**

- заставляет браузер использовать HTTPS;
- снижает риск downgrade-атак;
- помогает защититься от MITM;
- может применяться ко всем поддоменам.

---

### Настройка Referrer-Policy

```typescript
import { FABShield } from '@fab-orbita/shield'

const shield = new FABShield({
  headers: {
    referrerPolicy: {
      enabled: true,
      policy: 'strict-origin-when-cross-origin'
    }
  }
})
```

---

### Доступные политики Referrer-Policy

| Политика | Описание |
|:---|:---|
| `no-referrer` | Не отправлять referrer |
| `no-referrer-when-downgrade` | Не отправлять referrer при переходе HTTPS → HTTP |
| `same-origin` | Отправлять referrer только на том же origin |
| `origin` | Отправлять только схему, хост и порт |
| `strict-origin` | Отправлять origin только при безопасном переходе |
| `origin-when-cross-origin` | Полный referrer на том же origin, только origin на других |
| `strict-origin-when-cross-origin` | Безопасная политика по умолчанию для большинства проектов |
| `unsafe-url` | Отправлять полный URL всегда, обычно не рекомендуется |

## 🛡️ Настройка CSP

**CSP** — Content Security Policy. Это один из главных механизмов защиты от XSS и неразрешенных источников ресурсов.

---

### Базовая конфигурация CSP

```typescript
import { FABShield } from '@fab-orbita/shield'

const shield = new FABShield({
  csp: {
    // Включить CSP
    enabled: true,

    // Режим отчетности без блокировки
    reportOnly: false,

    // Динамический CSP
    dynamic: true,

    // Доверенные CDN
    trustedCDNs: [
      'https://cdnjs.cloudflare.com',
      'https://cdn.jsdelivr.net',
      'https://unpkg.com'
    ],

    // Кастомные директивы
    directives: {
      'default-src': ["'self'"],
      'script-src': ["'self'"],
      'style-src': ["'self'", "'unsafe-inline'"],
      'img-src': ["'self'", 'data:', 'https:'],
      'font-src': ["'self'", 'https:', 'data:'],
      'connect-src': ["'self'", 'https://api.example.com'],
      'frame-ancestors': ["'none'"],
      'base-uri': ["'self'"],
      'form-action': ["'self'"]
    }
  }
})
```

---

### Расширенная конфигурация CSP

```typescript
import { FABShield } from '@fab-orbita/shield'

const shield = new FABShield({
  csp: {
    enabled: true,
    dynamic: true,
    reportOnly: false,

    // Отчеты о нарушениях
    reportUri: '/api/csp-report',
    reportTo: 'csp-endpoint',

    // Динамические директивы
    dynamicDirectives: {
      'script-src': (req) => {
        const nonce = generateNonce()
        return [`'nonce-${nonce}'`, "'self'"]
      },

      'style-src': (req) => {
        return ['https:', "'unsafe-inline'"]
      }
    },

    // Доверенные источники
    trustedCDNs: [
      'https://cdnjs.cloudflare.com',
      'https://cdn.jsdelivr.net'
    ],

    // Разрешить nonce для inline-скриптов
    nonceEnabled: true,

    // Разрешить inline-стили
    unsafeInlineStyles: false,

    // Строгий режим
    strict: true
  }
})
```

---

### Рекомендации по CSP

| Сценарий | Рекомендация |
|:---|:---|
| Production API | Используйте строгий CSP |
| SPA-приложение | Разрешайте только нужные CDN |
| SSR-приложение | Используйте nonce |
| Legacy-проект | Начните с `reportOnly: true` |
| Админ-панель | Используйте максимально строгие директивы |

## 🔌 Настройка плагинов

Плагины позволяют расширять FAB Shield под конкретные задачи проекта.

```typescript
import { FABShield } from '@fab-orbita/shield'

const shield = new FABShield({
  plugins: [
    // WAF-интеграция
    {
      name: 'waf-integration',
      enabled: true,
      config: {
        provider: 'cloudflare',
        apiKey: process.env.CLOUDFLARE_API_KEY
      },

      middleware: async (req, res, next) => {
        // Логика интеграции с WAF
        next()
      }
    },

    // Geo-блокировка
    {
      name: 'geo-blocking',
      enabled: true,
      config: {
        blockedCountries: ['XX', 'YY']
      },

      middleware: async (req, res, next) => {
        const country = await getCountry(req.ip)
        const blockedCountries = ['XX', 'YY']

        if (blockedCountries.includes(country)) {
          res.status(403).send('Access denied')
          return
        }

        next()
      }
    },

    // Аудит
    {
      name: 'audit-logger',
      enabled: true,
      config: {
        logPath: './logs/audit.log',
        includeHeaders: false,
        includeBody: false
      },

      middleware: (req, res, next) => {
        console.log(`${new Date().toISOString()} ${req.method} ${req.url}`)
        next()
      }
    }
  ]
})
```

## 📝 Настройка кастомных правил

Кастомные правила позволяют описывать собственные условия обработки запросов.

```typescript
import { FABShield } from '@fab-orbita/shield'

const shield = new FABShield({
  rules: [
    // Блокировка по IP
    {
      path: '*',
      action: 'block',
      condition: (req) => {
        const blockedIPs = ['192.168.1.100', '10.0.0.50']
        return blockedIPs.includes(req.ip)
      }
    },

    // Ограничение по методу
    {
      path: '/api/*',
      method: 'DELETE',
      action: 'block',
      condition: (req) => !req.headers['x-admin-token']
    },

    // Защита админки
    {
      path: '/admin/*',
      action: 'challenge',
      condition: (req) => {
        return !req.session?.isAdmin
      }
    },

    // Rate limiting для API
    {
      path: '/api/*',
      action: 'rate-limit',
      condition: (req) => true,
      config: {
        maxRequests: 100,
        windowMs: 60000
      }
    }
  ]
})
```

## 📋 Настройка логирования

```typescript
import { FABShield } from '@fab-orbita/shield'

const shield = new FABShield({
  logging: {
    // Уровень логирования
    level: 'info',

    // Формат логов
    format: 'json',

    // Куда писать
    transports: [
      {
        type: 'console',
        enabled: true
      },

      {
        type: 'file',
        enabled: true,
        path: './logs/shield.log',
        maxSize: '10mb',
        maxFiles: 5
      },

      {
        type: 'remote',
        enabled: false,
        url: 'https://logs.example.com'
      }
    ],

    // Что логировать
    include: {
      requests: true,
      threats: true,
      errors: true,
      performance: true,
      metrics: true
    },

    // Исключить чувствительные данные
    exclude: {
      headers: ['Authorization', 'Cookie'],
      body: ['password', 'token', 'refreshToken']
    }
  }
})
```

| `info` | Основные события |

## ✅ Production-чеклист

Перед запуском в production проверьте:

- [ ] Используется `env: 'production'`
- [ ] Включен HTTPS
- [ ] Настроен HSTS
- [ ] CSP проверен в `reportOnly` перед блокировкой
- [ ] Секреты не хранятся в коде
- [ ] API-ключи лежат в environment variables
- [ ] Включено исключение чувствительных данных из логов
- [ ] Настроен rate limiting
- [ ] Настроен мониторинг
- [ ] Настроены алерты
- [ ] Проверены кастомные правила
- [ ] Обновлены зависимости
- [ ] Приложение протестировано на staging

## ⚠️ Важные рекомендации

### Не храните секреты в коде

Плохо:

```typescript
const shield = new FABShield({
  plugins: [
    {
      name: 'waf-integration',
      config: {
        apiKey: 'real-secret-key'
      }
    }
  ]
})
```

Хорошо:

```typescript
const shield = new FABShield({
  plugins: [
    {
      name: 'waf-integration',
      config: {
        apiKey: process.env.WAF_API_KEY
      }
    }
  ]
})
```

---

### Начинайте с мягкого режима

Для новых production-проектов полезно сначала включить мониторинг и `reportOnly`, а затем постепенно включать блокировку.

```typescript
const shield = new FABShield({
  csp: {
    enabled: true,
    reportOnly: true
  },

  ai: {
    enabled: true,
    blocking: {
      enabled: false
    }
  }
})
```

---

### Не логируйте чувствительные данные

```typescript
const shield = new FABShield({
  logging: {
    exclude: {
      headers: ['Authorization', 'Cookie', 'Set-Cookie'],
      body: ['password', 'token', 'refreshToken', 'secret']
    }
  }
})
```
