## 📋 Введение

**FAB Shield** можно установить несколькими способами:

- через `npm`;
- через `yarn`;
- через `pnpm`;
- через **Fab Registry**;
- внутри Docker-контейнера.

В этом руководстве описаны системные требования, установка, проверка, настройка окружения и устранение типичных проблем.

## 🚀 Способы установки

## 1. Установка через npm

Основной способ установки:

```bash
npm install @fab-orbita/shield
```

Для production-зависимости:

```bash
npm install @fab-orbita/shield --save-prod
```

Для dev-зависимости:

```bash
npm install @fab-orbita/shield --save-dev
```

---

## 2. Установка через yarn

```bash
yarn add @fab-orbita/shield
```

Для dev-зависимости:

```bash
yarn add @fab-orbita/shield --dev
```

---

## 3. Установка через pnpm

```bash
pnpm add @fab-orbita/shield
```

Для dev-зависимости:

```bash
pnpm add -D @fab-orbita/shield
```

---

## 4. Установка через Fab Registry

Установка напрямую из Fab Registry:

```bash
npm install @fab-orbita/shield --registry=https://fab.devorbit.ru
```

Или настройте scope для Fab Registry:

```bash
npm config set @fab-orbita:registry https://fab.devorbit.ru
npm install @fab-orbita/shield
```

Проверить текущие настройки npm registry:

```bash
npm config get registry
npm config get @fab-orbita:registry
```

## ⚙️ Настройка окружения

### 1. Создание `.env` файла

```env
NODE_ENV=production

SHIELD_AI_ENABLED=true
SHIELD_MONITORING_ENABLED=true
SHIELD_LOG_LEVEL=info

SHIELD_RATE_LIMIT_MAX=100
SHIELD_RATE_LIMIT_WINDOW_MS=60000
```

> Не храните реальные секреты в репозитории. Добавьте `.env` в `.gitignore`.

---

### 2. Настройка TypeScript

Пример `tsconfig.json`:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "types": ["node"],
    "outDir": "dist"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

Для ESM-проекта можно использовать:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "types": ["node"],
    "outDir": "dist"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

---

### 3. Настройка ESLint

Пример `.eslintrc.json`:

```json
{
  "root": true,
  "parser": "@typescript-eslint/parser",
  "plugins": ["@typescript-eslint"],
  "extends": [
    "eslint:recommended",
    "plugin:@typescript-eslint/recommended"
```

Установка зависимостей для ESLint:

```bash
npm install -D eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
```

## 🧪 Минимальный пример после установки

```typescript
import express from 'express'
import { FABShield } from '@fab-orbita/shield'

const app = express()
const shield = new FABShield()

app.use(shield.middleware())

app.get('/', (req, res) => {
  res.json({
    message: 'FAB Shield работает!'
  })
})

app.listen(3000, () => {
  console.log('🚀 Server started on http://localhost:3000')
})
```

Установить Express:

```bash
npm install express
npm install -D @types/express tsx typescript
```

Запуск:

```bash
npx tsx app.ts
```

## 🐳 Установка в Docker

### Dockerfile

```dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./

RUN npm ci

COPY . .

ENV NODE_ENV=production

EXPOSE 3000

CMD ["node", "dist/app.js"]
```

---

### package.json для Docker

```json
{
  "name": "fab-shield-app",
  "version": "1.0.0",
  "type": "commonjs",
  "scripts": {
    "build": "tsc",
    "start": "node dist/app.js"
  },
  "dependencies": {
    "@fab-orbita/shield": "^1.0.0",
    "express": "^4.18.0"
  },
  "devDependencies": {
    "@types/express": "^4.17.0",
    "@types/node": "^20.0.0",
    "typescript": "^5.0.0"
  }
}
```

---

### Сборка образа

```bash
docker build -t fab-shield-app .
```

---

### Запуск контейнера

```bash
docker run -p 3000:3000 fab-shield-app
```

---

### docker-compose.yml

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      NODE_ENV: production
      SHIELD_AI_ENABLED: "true"
      SHIELD_MONITORING_ENABLED: "true"
    volumes:
      - ./logs:/app/logs
```

Запуск:

```bash
docker compose up --build
```

## 🚨 Устранение проблем

### Проблема: ошибка при установке

Очистите npm cache:

```bash
npm cache clean --force
```

Удалите `node_modules` и lock-файл:

```bash
rm -rf node_modules package-lock.json
```

Установите зависимости заново:

```bash
npm install
```

---

### Проблема: конфликт версий

Обновите зависимости:

```bash
npm update
```

Или установите конкретную версию:

```bash
npm install @fab-orbita/shield@1.0.0
```

---

### Проблема: TypeScript ошибки

Проверьте `tsconfig.json`:

```json
{
  "compilerOptions": {
    "skipLibCheck": true,
    "types": ["node"],
    "esModuleInterop": true
  }
}
```

Проверьте наличие типов Node.js:

```bash
npm install -D @types/node
```

---

### Проблема: registry недоступен

Используйте официальный npm registry:

```bash
npm install @fab-orbita/shield --registry=https://registry.npmjs.org
```

Или сбросьте registry:

```bash
npm config delete @fab-orbita:registry
npm config set registry https://registry.npmjs.org
```

---

### Проблема: пакет не найден

Проверьте правильное имя пакета:

```bash
npm install @fab-orbita/shield
```

Правильно:

```text
@fab-orbita/shield
```

Неправильно:

```text
@fab-registry/shield
```

### Проверка HTTP-заголовков

Запустите приложение и выполните:

```bash
curl -I http://localhost:3000
```

В ответе должны появиться security-заголовки, например:

```text

Strict-Transport-Security: ...
X-Frame-Options: ...
X-Content-Type-Options: nosniff
Referrer-Policy: ...
```
