## 🔒 Конфиденциальность

### Сбор данных

По умолчанию **FAB Shield** не собирает:

- ❌ Персональные данные
- ❌ Данные пользователей
- ❌ Информацию о проектах
- ❌ Ключи доступа
- ❌ Пароли
- ❌ Токены авторизации

---

### Логирование

**FAB Shield** может вести логи для технических целей:

- 🔍 Отладка
- 📊 Мониторинг
- 🛡️ Безопасность
- 🚨 Анализ инцидентов

Логи могут содержать:

| Тип данных | Описание |
|:---|:---|
| **IP-адреса** | Используются для анализа запросов и защиты от атак |
| **Пути запросов** | Используются для отладки и мониторинга |
| **Время запросов** | Используется для анализа производительности |
| **Статусы ответов** | Используются для диагностики ошибок |

---

### Настройка безопасного логирования

```typescript
const shield = new FABShield({
    logging: {
        level: 'info',
        exclude: {
            headers: ['Authorization', 'Cookie'],
            body: ['password', 'token']
        }
    }
})
```

Если FAB Shield используется в инфраструктуре, где проходят платежные данные, ответственность за соответствие PCI DSS несет владелец такой инфраструктуры.
