## 📋 Введение

**FAB Shield** — это инструмент для всех, кто разрабатывает веб-приложения, API и backend-сервисы на Node.js.

Но есть категории разработчиков, компаний и проектов, которым такой инструмент нужен **особенно сильно**: там, где есть пользовательские данные, авторизация, платежи, личные кабинеты, административные панели, API и требования к безопасности.

Этот документ поможет понять, подходит ли FAB Shield для ваших задач.

## 🎯 Кому FAB Shield может быть не нужен

### 1. 📄 Простые статические сайты

Примеры:

- простые HTML-страницы;
- сайты-визитки без backend;
- лендинги без форм и авторизации;
- полностью статические блоги.

**Почему:** если нет backend, API, авторизации и обработки пользовательских данных, полноценный Node.js security middleware может быть избыточным.

---

### 2. 🧪 Локальные тестовые проекты

Примеры:

- одноразовые прототипы;
- локальные эксперименты;
- учебный код без публикации;
- временные тестовые окружения.

**Почему:** если проект не выходит в production, риски ниже. Но для обучения безопасности FAB Shield все равно может быть полезен.

---

### 3. 🎮 Приложения без HTTP backend

Примеры:

- десктопные приложения без web API;
- мобильные игры без Node.js backend;
- embedded-системы;
- проекты на других протоколах.

**Почему:** FAB Shield ориентирован на HTTP/Web API и Node.js middleware.

## 💡 Примеры использования

### Enterprise-компания

```typescript
import { FABShield } from '@fab-orbita/shield'

// Банковское или корпоративное приложение
const shield = new FABShield({
  ai: {
    enabled: true,
    anomalyDetection: true,
    threatPrediction: true
  },

  monitoring: {
    enabled: true,
    alerting: true,
    audit: true
  },

  csp: {
    strict: true
  },

  rateLimit: {
    enabled: true,
    windowMs: 60_000,
    max: 100
  }
})
```

import { FABShield } from '@fab-orbita/shield'

// SaaS-платформа

## ✅ Когда точно стоит использовать

FAB Shield стоит использовать, если в вашем проекте есть:

- авторизация;
- личные кабинеты;
- API;
- формы;
- платежи;
- админ-панель;
- пользовательские данные;
- интеграции с внешними сервисами;
- публичные endpoints;
- риск brute force;
- необходимость security headers;
- требования к логированию и мониторингу.

---

## ⚠️ Когда нужно дополнительное решение

FAB Shield усиливает безопасность Node.js-приложения, но не заменяет:

- внешний WAF;
- аудит кода;
- pentest;
- DevSecOps-процессы;
- защиту инфраструктуры;
- правильную работу с секретами;
- защиту базы данных;
- юридический аудит соответствия требованиям;
- безопасную архитектуру всего проекта.
