## 🔒 Основные рекомендации

### 1. Всегда используйте HTTPS

**Почему:** HTTPS шифрует весь трафик между клиентом и сервером.

```typescript
// Включение HSTS
const shield = new FABShield({
    headers: {
        hsts: {
            maxAge: 31536000,
            includeSubDomains: true,
            preload: true
        }
    }
})
Дополнительно:

Используйте сертификаты Let's Encrypt

Настройте автоматическое обновление

Проверяйте валидность сертификатов

2. Включите все security-заголовки
Почему: Security-заголовки — первая линия защиты.

typescript
const shield = new FABShield({
    headers: {
        enabled: true,
        custom: {
            'X-Frame-Options': 'DENY',
            'X-Content-Type-Options': 'nosniff',
            'X-XSS-Protection': '1; mode=block',
            'Referrer-Policy': 'strict-origin-when-cross-origin'
        }
    }
})
3. Настройте CSP
Почему: CSP защищает от XSS атак.

typescript
const shield = new FABShield({
    csp: {
        enabled: true,
        dynamic: true,
        directives: {
            'default-src': ["'self'"],
            'script-src': ["'self'", "'strict-dynamic'"],
            'style-src': ["'self'", "'unsafe-inline'"],
            'img-src': ["'self'", "data:", "https:"]
        }
    }
})
Рекомендации:

Не используйте 'unsafe-inline' без необходимости

Добавляйте только доверенные CDN

Используйте nonce для инлайн-скриптов

4. Включите AI-защиту
Почему: AI обнаруживает новые угрозы.

typescript
const shield = new FABShield({
    ai: {
        enabled: true,
        anomalyDetection: true,
        threatPrediction: true,
        modules: {
            xssProtection: true,
            sqlInjectionProtection: true,
            ipReputation: true
        }
    }
})
5. Настройте Rate Limiting
Почему: Защита от DDoS и брутфорса.

typescript
const shield = new FABShield({
    rateLimit: {
        enabled: true,
        default: {
            windowMs: 60000,
            max: 100
        },
        paths: {
            '/api/auth/login': {
                windowMs: 60000,
                max: 5
            }
        }
    }
})
🔧 Конфигурация
6. Используйте переменные окружения
Почему: Безопасное хранение секретов.

bash
# .env
SHIELD_AI_ENABLED=true
SHIELD_RATE_LIMIT_MAX=100
REDIS_PASSWORD=secure_password
typescript
const shield = new FABShield({
    ai: {
        enabled: process.env.SHIELD_AI_ENABLED === 'true'
    },
    rateLimit: {
        max: parseInt(process.env.SHIELD_RATE_LIMIT_MAX || '100')
    }
})
7. Настройте логирование
Почему: Для аудита и отладки.

typescript
const shield = new FABShield({
    logging: {
        level: 'info',
        format: 'json',
        transports: [
            {
                type: 'file',
                path: './logs/shield.log',
                maxSize: '10mb',
                maxFiles: 5
            }
        ]
    }
})
8. Включите мониторинг
Почему: Для отслеживания угроз.

typescript
const shield = new FABShield({
    monitoring: {
        enabled: true,
        export: ['prometheus'],
        alerts: {
            enabled: true,
            rules: [
                {
                    name: 'High Threat Rate',
                    metric: 'threats_blocked',
                    threshold: 10,
                    severity: 'critical'
                }
            ]
        }
    }
})

10. Используйте безопасные пароли
Почему: Защита от брутфорса.

Рекомендации:

Длина: минимум 12 символов

Символы: буквы, цифры, спецсимволы

Уникальность: не использовать повторно

Регулярная смена

11. Включите 2FA
Почему: Дополнительный уровень защиты.

typescript
// Включение 2FA для администраторов
const shield = new FABShield({
    ai: {
        modules: {
            twoFactorCheck: true
        }
    },
    rules: [
        {
            path: '/admin/*',
            require2FA: true,
            action: 'challenge'
        }
    ]
})

🚀 Продвинутые рекомендации
13. Создавайте кастомные правила
Почему: Для специфических угроз.

typescript
const shield = new FABShield({
    rules: [
        {
            name: 'Block specific IP',
            path: '*',
            action: 'block',
            condition: (req) => req.ip === '1.2.3.4'
        },
        {
            name: 'Block specific user-agent',
            path: '*',
            action: 'block',
            condition: (req) => req.headers['user-agent']?.includes('bot')
        }
    ]
})
14. Используйте плагины
Почему: Расширение функциональности.

typescript
const shield = new FABShield({
    plugins: [
        {
            name: 'geo-blocking',
            middleware: (req, res, next) => {
                // Гео-блокировка
                const country = getCountry(req.ip)
                if (blockedCountries.includes(country)) {
                    res.status(403).send('Access denied')
                    return
                }
                next()
            }
        }
    ]
})
