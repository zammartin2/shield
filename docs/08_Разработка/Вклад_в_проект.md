```bash

# Добавьте upstream
```

### Код-стайл

```typescript
// ✅ ПРАВИЛЬНО
export async function processRequest(
    request: Request,
    options: ProcessOptions
): Promise<Result> {
    // Проверяем входные данные
    if (!request) {
        throw new Error('Request is required')
    }

    // Обрабатываем
    const result = await doWork(request, options)

    // Возвращаем
    return result
}

// ❌ НЕПРАВИЛЬНО
export async function process(req, opt) {
    if(!req) throw "req required"
    let r=await doWork(req,opt)
    return r
}
```

## 🧪 Написание тестов

### Структура тестов

```typescript
// src/__tests__/module.test.ts

import { describe, test, expect, beforeEach, afterEach } from '@jest/globals'

describe('Module Name', () => {
    let instance

    beforeEach(() => {
        // Подготовка
        instance = new Module()
    })

    afterEach(() => {
        // Очистка
        instance.destroy()
    })

    test('should work correctly', () => {
        // Arrange
        const input = 'test'

        // Act
        const result = instance.process(input)

        // Assert
        expect(result).toBe('expected')
    })

    test('should handle errors', () => {
        // Arrange
        const input = null

        // Act & Assert
        expect(() => instance.process(input)).toThrow()
    })
})
```

## 🔍 Код-ревью

### Что проверяем

| Критерий | Описание |
|:---|:---|
| ✅ **Функциональность** | Работает ли код |
| ✅ **Качество** | Читаемость, стиль |
| ✅ **Тесты** | Есть ли тесты |
| ✅ **Документация** | Обновлена ли документация |
| ✅ **Производительность** | Нет ли регрессий |
| ✅ **Безопасность** | Нет ли уязвимостей |
| ✅ **Совместимость** | Не сломает ли изменение обратную совместимость |
