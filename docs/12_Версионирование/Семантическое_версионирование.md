## 📌 Внутренние правила

| Тип версии | Формат | Пример | Условия |
|------------|--------|--------|---------|
| **Major** | `x.0.0` | `2.0.0` | Критические изменения API |
| **Minor** | `1.x.0` | `1.4.0` | Новая функциональность |
| **Patch** | `1.3.x` | `1.3.2` | Баг-фиксы, безопасность |

---

## 📋 Процесс утверждения версий

1. **Инициация** — разработчик предлагает изменения
2. **Техническая экспертиза** — код-ревью, тесты
3. **Безопасность** — проверка на уязвимости
4. **Утверждение** — Tech Lead / CTO
5. **Релиз** — публикация в npm / Fab Registry
